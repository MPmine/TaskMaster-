# TaskMaster-
Project WebSite+API(Алиса)
TaskMaster — веб‑приложение для персонального управления задачами.


TaskMaster — это веб‑приложение на базе Flask, предназначенное для организации и контроля личных задач. Система позволяет пользователям регистрироваться, авторизовываться и управлять списком задач: создавать, просматривать, редактировать и отмечать выполненные.

3. Цель проекта
Создать интуитивно понятный инструмент для персонального тайм‑менеджмента, который:

помогает структурировать задачи;

повышает личную продуктивность;

даёт наглядное представление о текущих и выполненных делах;

обеспечивает безопасность данных через систему авторизации.

4. Основные функции
Регистрация и авторизация пользователей:

создание учётной записи с именем пользователя и email;

вход в систему с проверкой пароля;

защита маршрутов для неавторизованных пользователей.

Управление задачами:

добавление новой задачи (заголовок, описание, статус, приоритет, срок выполнения);

просмотр списка задач на дашборде;

фильтрация и визуальное отображение статуса задач («В работе» / «Завершена»);

индикация приоритета (низкий, средний, высокий).

Личный профиль:

просмотр данных учётной записи (имя пользователя, email);

возможность выхода из системы.

5. Ключевые особенности
Интуитивный интерфейс:

простая навигация по разделам;

адаптивный дизайн для разных устройств.

Безопасность:

хеширование паролей перед сохранением в БД;

CSRF‑защита форм;

ограничение доступа к персональным данным.

Наглядность:

цветовые индикаторы статуса и приоритета задач;

отображение сроков выполнения;

понятные сообщения об ошибках и успехах операций.

Персонализация:

дашборд с задачами текущего пользователя;

профиль с персональными данными.

6. Технологический стек
Бэкенд:

Python 3.x;

Flask (веб‑фреймворк);

Flask‑Login (аутентификация);

Flask‑WTF (формы);

SQLAlchemy (ORM для работы с БД).

Фронтенд:

HTML5;

CSS3 (стилизация интерфейса);

Jinja2 (шаблонизация).

База данных: SQLite (для прототипирования и развёртывания).

Дополнительные инструменты: Werkzeug (WSGI‑утилиты).

7. Архитектура приложения
Маршрутизация (Flask routes):

/ — главная страница;

/register — регистрация;

/login — авторизация;

/logout — выход;

/dashboard — дашборд с задачами;

/tasks — полный список задач;

/new_task — создание новой задачи;

/profile — личный профиль.

Модели данных (SQLAlchemy):

User (пользователь): id, username, email, password_hash;

Task (задача): id, title, description, status (pending/completed), priority (1–3), due_date, user_id (связь с пользователем).

Шаблоны (Jinja2):

base.html — базовый шаблон с навигацией;

index.html — главная страница;

register.html / login.html — формы регистрации и входа;

dashboard.html / tasks.html — отображение задач;

new.html — форма создания задачи;

profile.html — страница профиля.

Формы (Flask‑WTF):

RegistrationForm (имя, email, пароль, подтверждение пароля);

LoginForm (email, пароль);

TaskForm (название, описание, статус, приоритет, срок).

8. Целевая аудитория
частные пользователи, заинтересованные в личном тайм‑менеджменте;

фрилансеры и удалённые работники, которым нужно отслеживать задачи;

небольшие команды (с доработкой функционала для совместной работы).

9. Критерии успеха
Функциональность:

все заявленные функции работают без ошибок;

формы валидируют данные и показывают понятные сообщения об ошибках.

Производительность:

время отклика страниц — не более 2 с;

приложение стабильно работает при 50+ одновременных пользователях (на тестовом стенде).

Удобство использования:

интуитивно понятная навигация;

адаптивность интерфейса для мобильных устройств;

доступность основных функций в 1–2 клика.

Безопасность:

пароли хранятся в хешированном виде;

CSRF‑токены защищают формы;

неавторизованные пользователи не получают доступ к персональным данным.

10. Планы развития
Краткосрочные (1–2 месяца):

добавить фильтрацию и сортировку задач (по статусу, приоритету, дате);

реализовать редактирование и удаление задач;

улучшить дизайн с помощью UI‑фреймворка (например, Bootstrap).

Среднесрочные (3–6 месяцев):

внедрить уведомления (напоминания о сроках);

добавить категории/теги для задач;

разработать API для мобильного приложения.

Долгосрочные (6+ месяцев):

поддержка командной работы (общие проекты, роли);

интеграция с календарями (Google Calendar и т. д.);

экспорт задач в CSV/PDF.

11. Ограничения и зависимости
Ограничения:

текущая версия ориентирована на одиночных пользователей;

БД SQLite подходит для прототипирования, но для продакшена рекомендуется PostgreSQL/MySQL.

Зависимости (requirements.txt):

Flask;

Flask‑Login;

Flask‑SQLAlchemy;

Flask‑WTF;

WTForms.
